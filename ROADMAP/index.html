Your Arsenal

3 Books. Strategically Mapped.

You already have the right books. Here's exactly when and how to use each one.

📗
Phase 1 + 3
Java: A Beginner's Guide
— Herbert Schildt
Start here on Day 1. This is your primary companion through Phase 1. Schildt's explanations are crystal clear — read it like a novel, chapter by chapter, not as a reference. Return to the advanced chapters in Phase 3.
Read in Phase 1
Ch 1–5: Syntax Ch 6–8: OOP Ch 9–11: Generics Ch 12–14: Collections
Return in Phase 3
Ch 15: JDBC Ch 18: Lambdas Ch 20–22: I/O
📘
Phase 2 — DSA
Introduction to Java Programming
— Y. Daniel Liang (9th Ed.)
Your DSA bible. Chapters 20–28 are some of the best data structure explanations written in Java. Liang traces through every algorithm visually with diagrams. Skip Chapter 32 (JavaFX) unless you want GUI development.
Core DSA Chapters
Ch 20: Lists/Stacks/Queues Ch 21: Sets/Maps Ch 22: Efficient Algos Ch 23–25: Sorting/BST/AVL Ch 26: Hashing Ch 27–28: Graphs
📙
Parallel Read
Introduction to Programming Using Java
— David Eck (javanotes5.pdf · Free)
Read alongside Schildt in Phase 1. Eck is a professor and explains why things work, not just how. His OOP and linked data structures chapters build the deepest mental models. Available free at math.hws.edu.
Strongest Chapters
Ch 1–4: Foundations Ch 5–8: OOP (best in class) Ch 9: Linked Structures Ch 10–11: I/O & Generics
💡
Reading order: Start Schildt (ch 1–10) + Eck (ch 1–5) together in Month 1. Switch to Liang for Phase 2 DSA. Come back to Schildt ch 15+ for JDBC and advanced Java in Phase 3.

Daily Routine

How to Structure Your Day

Adapt to your schedule. The key: DSA and concept study should never share the same mental block.

Phases 1–2: Weekday (Learning Mode)
6:00–7:30
Theory reading — 1 chapter from your current book, take notes
9:00–12:00
Code implementation — implement what you read, write from scratch
13:00–15:00
LeetCode — 2 problems. 1 easy warm-up + 1 medium focus
20:00–21:30
Review + revision — revisit yesterday's solutions, understand why
Phases 3–4: Weekday (Builder Mode)
6:00–7:00
LeetCode warm-up — 1 medium problem (maintain streak)
9:00–13:00
Spring Boot / Dev project — build a feature, debug, test
14:00–16:00
YouTube / Docs — Spring guide or system design video
20:00–21:30
Project polish — README, Postman, deployment
